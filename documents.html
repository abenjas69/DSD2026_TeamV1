MEETING NOTE

S2 Interface Alignment

Record of the V1 and S2 discussion about IF1 data requirements and expected comparison values.

V1 request to S2

V1 explained that the motion-recognition pipeline ideally needs the following information from S2:

  • Measured sensor data or motion stream.
  • Timestamp or sequence information.
  • Sensor or joint identification, if available.
  • Payload format and units defined in IF1.
  • Payload status and error information.
  • Target Angle or expected values used for comparison.

S2 response

S2 confirmed that they can provide the requested sensor-side data. According to their analysis, the expected Target Angle value may not be provided by S2 and may be stored in the V2 database.

  • Confirmed by S2 Measured data, timing or sequence information, identifiers when available, IF1 format and units, and payload status or error information can be provided.
  • Open dependency Target Angle / expected values must be aligned with V2 or the agreed server workflow.

Follow-up for V1

  • IF1 Keep the S2 -> V1 contract focused on motion payload structure, timing/order, sensor or joint identifiers, units, status, and error fields.
  • V1-V2 Confirm with V2 where Target Angle / expected values are stored and how V1 can access them for comparison.
  • Comparison logic Treat missing expected values as a partial comparison case until the source of truth is agreed.
